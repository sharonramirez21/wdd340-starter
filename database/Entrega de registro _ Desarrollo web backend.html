Entregar vista de registro

Introducción

En la actividad anterior, utilizó el proceso MVC para habilitar el enlace "Mi cuenta" y entregó exitosamente elAccesoVista al cliente. Esta vista contenía un formulario de inicio de sesión y un enlace para solicitar la vista de registro y crear una cuenta. Esta actividad consiste en hacer que el enlace para solicitar la vista de registro funcione.

Descripción general del video

El video ofrece una descripción general de la actividad, pero no contiene los detalles necesarios para completar cada proceso. Vea el video para obtener una idea general, pero siga los pasos escritos para completar la actividad. Esta es la transcripción del video.

La vista de inicio de sesión

Comenzarás esta actividad donde terminó la última: en elaccesoVista. ¿Por qué? Porque se debe asignar una ruta al enlace y esta debe coincidir con el formato general de la ruta "Mi cuenta".

Recordarás que la ruta en el enlace "Mi cuenta" se veía así:

href="/account/login".
El accountelemento fue colocado en elservidor.jsarchivo como el primer elemento de la función app.use(), mientras que "login" era parte del archivo del enrutador, para indicar el proceso real que se ejecutaría, si se detectaba en el objeto GET.Comprender la ruta y cómo encaja en el proceso MVC es fundamental para el siguiente paso.Las dos partes del camino, como se mencionó, se muestran aquí:

  1. Examine atentamente la ruta href del enlace de registro (si hay alguno) debajo del formulario de inicio de sesión.
  2. Agregue o modifique la ruta href (si es necesario) para que se parezca a la ruta "Mi cuenta", pero indique que se está solicitando la vista "registro".
  3. Guarde el archivo.

La ruta de registro

Es hora de agregar una nueva ruta a laRuta de la cuentaarchivo, para iniciar la entrega de la vista de registro.

  1. Abrir elRuta de la cuentaarchivo en elrutascarpeta.
  2. Usando la ruta de inicio de sesión como ejemplo (consulte el ejemplo anterior), cree una nueva registrationruta.
  3. Como antes, la ruta debe activar una función, que se encuentra en elcontrolador de cuenta, que necesitarás construir.
  4. Asegúrese de que VSC no muestre advertencias ni errores .
  5. Guarde el archivo.

Controlador de cuentas

Con la ruta de registro establecida, es necesario construir la función del controlador de cuenta para procesar la solicitud de vista de registro.

  1. Encuentra y abre elControlador de cuentasarchivo en elcontroladorescarpeta.
  2. Agregue algunas líneas vacías debajo de la función existente para mostrar la vista de inicio de sesión. Luego, agregue un comentario y una función que mostrará la vista de registro (la creará más adelante en esta actividad).
  3. La función de registro se muestra a continuación (Sus nombres pueden ser diferentes a lo que se muestra.): 
    /* ****************************************
*  Deliver registration view
* *************************************** */
async function buildRegister(req, res, next) {
  let nav = await utilities.getNav()
  res.render("account/register", {
    title: "Register",
    nav,
    errors: null
  })
}

module.exports = { buildLogin, buildRegister }

  4. Una explicación

    • Líneas 1 a 3: un comentario de varias líneas que presenta la función.
    • Línea 4: comienza la función, declarándola como asincrónica y pasando solicitud, respuesta y siguiente como parámetros.
    • Línea 5: recupera y almacena la cadena de la barra de navegación para su uso en la vista.
    • Línea 6: llama a la función de renderizado e indica la vista que se devolverá al cliente y abre el objeto que llevará los datos a la vista. Nota: la ruta indica que la vista está dentro de la carpeta "cuenta", que debería estar dentro de la carpeta "vistas".
    • Líneas 7-8: los elementos de datos que se enviarán a la vista.
    • Línea 9: cierra el objeto de datos y la función de renderizado (abierta en la línea 6).
    • Línea 10: cierra la función.
    • Línea 11: se dejó en blanco intencionalmente.
    • Línea 12: exporta la función para su uso en otro lugar[Nota: esta función se agrega al objeto de exportaciones, el otro nombre de función ya debería existir]..
  5. Asegúrese de que VSC no muestre advertencias ni errores .
  6. Guarde el archivo.

Vista de registro

Con todo lo demás hecho, es hora de construir elregistroVista. Al igual que las demás vistas creadas, debe cumplir las siguientes especificaciones:

  1. Ser creado dentro de lacuentacarpeta, dentro de lapuntos de vistacarpeta.
  2. Se nombrará según el nombre en la función de renderizado del controlador.
  3. Utilice el valor de los datos del título tanto para el título como para los elementos h1.
  4. Tener la capacidad de mostrar un mensaje flash, si se envía uno.
  5. Muestra un formulario HTML de registro que acepta cuatro campos: nombre, apellidos, correo electrónico y contraseña. Cada campo debe ser del tipo correcto.Recuerde aplicar la técnica de seguimiento de datos de la última actividad también a estas entradas de formulario.
  6. El formulario debe incluir un botón para enviar los datos de registro. Vea este ejemplo:
  7. El formulario debe indicar que todos los datos son obligatorios y que la contraseña debe ser:
    • 12 caracteres de longitud, mínimo
    • contener al menos 1 letra mayúscula
    • contener al menos 1 número
    • contener al menos 1 carácter especial
    No es necesario hacer cumplir estos requisitos en este momento, ya que se agregarán más adelante.
  8. Opcionalmente, puede agregar un botón para cambiar la contraseña de viñetas a texto sin formato y viceversa. Siga el proceso que se muestra en este video para agregar esta función al formulario:

  9. Agregue CSS apropiado para representar la vista y el formulario de manera profesional.
  10. Asegúrese de que VSC no muestre advertencias ni errores .
  11. Guarde el archivo.

Tiempo de prueba

Bien, es importante probar la funcionalidad que se acaba de agregar.

  1. Asegúrese de que todos los nombres de archivos sean correctos, especialmente aquellos que requieren otros archivos.
  2. Abra una terminal VSC e inicie el servidor de desarrollo - pnpm run dev.
  3. Abra un navegador para localhost:5500.
  4. Haga clic en el enlace “Mi cuenta” en el encabezado.
  5. ElaccesoSe debe entregar la vista. Busque y haga clic en el enlace para solicitar la vista de registro.
  6. ElregistroLa vista debe ser entregada.
  7. Pruebe la vista contra la lista de verificación del frontend.
  8. Si todo funcionó, levanta el puño y grita "¡Hurra!". Si no, revisa los registros de la terminal para ver si se reporta algún error o pide ayuda a tu equipo de aprendizaje, al ayudante de cátedra o al profesor.
  9. ¡Detenga el servidor local cuando termine la prueba!
